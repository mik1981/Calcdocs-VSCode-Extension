CalcDocs Guida Completa

Formule YAML + calcoli inline per progetti C/C++ embedded.

1) Formule YAML: struttura

ADC_MAX:
  type: const
  value: 4095
  unit: count

R_PULLUP:
  type: const
  value: 10
  unit: kohm

NTC_ADC:
  type: expr
  expr: ADC_MAX * NTC_R / (R_PULLUP + NTC_R)
  unit: count
  value: auto

type: const valore scalare fisso

type: expr espressione con dipendenze

type: lookup valore tabellare da CSV (diretto o interpolato)

2) Grafo dipendenze e ordine di valutazione

CalcDocs costruisce un grafo delle dipendenze e risolve i simboli in ordine topologico.

  • Errore se una variabile è indefinita.
  • Errore se c’è una dipendenza circolare.
  • Warning se una formula dipende da simboli C/C++ condizionali ambigui.

3) Unità, compatibilità e controlli dimensionali

Unità compatibili si possono mescolare liberamente se la dimensione fisica coincide:

PRESSIONE_OK:
  type: expr
  expr: 100 bar + 10 kPa
  unit: atm    # catena valida bar -> Pa <- kPa -> atm

La conversione automatica viene applicata solo quando strettamente necessaria: somme/sottrazioni tra unità compatibili e conversioni finali tra varianti scalate della stessa unità (es. V -> mV). Nelle moltiplicazioni/divisioni pure cross-sistema (es. bar -> atm) viene mantenuta l'unità nativa del risultato.

Somme/sottrazioni incompatibili vengono bloccate:

SOMMA_ERRATA:
  type: expr
  expr: 10 ohm + 3 V   # errore

Se unit: non è definita, in hover viene mostrata l’unità risultante dalla dimensione calcolata (quando disponibile).

4) CSV / lookup nelle formule

NTC_R:
  type: lookup
  table: ntc_10k_table.csv
  row: 25
  lookupColumn: temp_c
  valueColumn: resistance_ohm
  interpolation: linear
  unit: ohm

Forma equivalente come espressione:

expr: csv("ntc_10k_table.csv", 25, "temp_c", "resistance_ohm", "linear")

5) Explain mode in hover

L’hover include:

  • valore calcolato e unità finale
  • formula espansa
  • simboli risolti (con valore + unità quando disponibili)
  • passaggi step-by-step
= 4095 * 10 / (10000 + 10)
= 4095 * 10 / 10010
= 4.0909090909

6) Calcoli inline (commenti C/C++)

// @vin = 12V
// @r = 4.7kOhm
// = @vin / @r -> mA

// = 25% * 200W -> W
// = 100 bar + 10 kPa -> atm
// = 3 ft + 12 in -> m

Inline supporta conversioni SI/anglosassoni compatibili e diagnostiche dimensionali.

7) Direttive ignore inline

// = A + B #calcdocs-ignore-warning
// = BAD_EXPR #calcdocs-ignore-error
// calcdocs-ignore-line; = qualunque cosa

Disponibili: calcdocs-ignore, calcdocs-ignore-error, calcdocs-ignore-warning, calcdocs-ignore-info, calcdocs-ignore-line.

8) Estrazione unità da C/C++

#define R_PULLUP 10000 // ohm
const float VREF = 3.3; // V

CalcDocs importa automaticamente le unità trovate e arricchisce i simboli YAML. I simboli mancanti vengono proposti in diagnostica.

9) Coesistenza hover con altre estensioni

L’hover CalcDocs è limitato ai contesti realmente CalcDocs (simboli/macros noti e inline), così da ridurre conflitti con IntelliSense e provider terze parti.

Se non c’è contesto CalcDocs, il provider non mostra hover e lascia invariata l’esperienza delle altre estensioni.

10) Workflow consigliato

  1. Definisci costanti in C/C++ con commento unità.
  2. Crea formule YAML con type, expr e unit opzionale.
  3. Usa hover/diagnostiche per correggere undefined, cicli e mismatch unitari.
  4. Usa inline per verifiche veloci direttamente nel sorgente.
