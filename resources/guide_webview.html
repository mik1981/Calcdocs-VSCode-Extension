Benvenuto in CalcDocs

CalcDocs è un'estensione VS Code per l'analisi di codice C/C++ con supporto a formule matematiche, propagazione dell'incertezza e simulazioni Monte Carlo.

Webview interattivaAlbero di dipendenze navigabile con slider

Come iniziare

  1. Aprire un workspace con file C/C++
  2. Creare un file formulas.yaml nella radice del progetto
  3. Definire simboli const ed expr con campi uncertainty e propagation
  4. Aprire la webview interattiva (comando CalcDocs: Open Interactive View)

Struttura file YAML

MY_VAR:
  type: const
  value: 100
  unit: mm
  uncertainty:
    type: percent
    value: 5        # ±5%
  distribution:
    type: normal
    sigma_level: 3

MY_FORMULA:
  type: expr
  formula: MY_VAR * 2
  unit: mm
  propagation: rss  # worst_case | rss | monte_carlo

Blocco uncertainty (input)

uncertainty:
  type: percent     # percent | range | absolute | sigma
  value: 5          # per type=percent: percentuale ±N%

uncertainty:
  type: range
  min: 95
  max: 105          # intervallo fisso

uncertainty:
  type: absolute
  absolute: 3       # ±3 in unità assolute

uncertainty:
  type: sigma
  sigma: 1.5        # ±3σ → halfWidth = 3*1.5 = 4.5

Blocco distribution (input)

distribution:
  type: uniform         # distribuzione uniforme (default)

distribution:
  type: normal
  sigma_level: 3        # σ = halfWidth / sigma_level (default 3)

distribution:
  type: triangular
  mode_value: 100       # valore più probabile

Propagazione output

propagation: worst_case    # somma dei |sensitivity|

propagation: rss           # sqrt(Σ(dfdx·σ)²) × 3

propagation: monte_carlo
confidence: 95             # percentile confidence (default 95)
seed: 42                   # seed PRNG (opzionale, per riproducibilità)
samples: 50000             # numero campioni (opzionale)

Override per parametro

MY_FORMULA:
  type: expr
  formula: BASE * FACTOR
  propagation: worst_case
  parameter_tolerances:
    BASE:
      uncertainty:
        type: percent
        value: 5
      distribution:
        type: uniform

Formato legacy (deprecato)

I campi legacy producono warning. Usare il formato moderno.
MY_VAR:
  type: const
  value: 100
  tol: 5              # → uncertainty.percent = 5
  tol_mode: gaussian  # → distribution.normal
  sigma: 3            # → distribution.sigma_level = 3

Modello di Incertezza degli Input

Ogni input ha due specifiche ortogonali: l'ampiezza dell'incertezza e la forma della distribuzione probabilistica.

Normalizzazione

Tutti i tipi di uncertainty vengono convertiti in NormalizedUncertainty:

{ nominal, lower, upper, halfWidth }
// halfWidth = (upper - lower) / 2
typeFormulahalfWidth
percent(N)lower = nom−|nom|×N/100|nom|×N/100
range(a,b)lower=min(a,b), upper=max(a,b)(b−a)/2
absolute(d)lower=nom−d, upper=nom+dd
sigma(σ)lower=nom−3σ, upper=nom+3σ3σ

Deviazione standard (per RSS e MC)

normal(sigma_level=k)halfWidth / k
uniformhalfWidth / √3 ≈ 0.577 × hw
triangularhalfWidth / √6 ≈ 0.408 × hw

Metodi di Propagazione dell'Incertezza

Worst Case

Somma lineare delle sensitività. Intervallo garantito al 100%. Conservativo.

RSS

Combinazione quadratica delle stddev. Probabilistico. sigmaOut=3 (fisso).

Monte Carlo

Campionamento diretto. Gestisce non-linearità e distribuzioni miste.

Worst Case — formula

per ogni input i:
  sensitivity_i = (f(nom + hw_i) - f(nom - hw_i)) / 2

Δ_total = Σ |sensitivity_i|
output  = [nominal − Δ_total, nominal + Δ_total]

RSS — formula

per ogni input i:
  dfdx_i       = (f(nom + hw_i) - f(nom - hw_i)) / (2 × hw_i)
  sensitivity_i = dfdx_i × σ_i    # σ_i = computeStdDev(norm, dist)

σ_output = √Σ(sensitivity_i²)
Δ        = 3 × σ_output           # sigmaOut = 3 (hardcoded)
output   = [nominal − Δ, nominal + Δ]
Nota: per funzione lineare f(x)=x con input normal(sl=3, tol=5%): sensitivity = dfdx × hw/3 = 1 × 5/3. Δ = 3 × 5/3 = 5 → stesso risultato del Worst Case. RSS dà range più ampio del WC solo con input uniform (σ=hw/√3 > hw/3).

Simulazione Monte Carlo

Il Monte Carlo campiona i valori degli input secondo le loro distribuzioni e valuta la formula per ogni campione. L'output è un istogramma della distribuzione risultante.

Algoritmo implementato

PRNG: SplitMix32 (seeding) → xoshiro128** (generazione)

N campioni:
  ≤ 5 input  → 100 000 campioni
  ≤ 20 input →  50 000 campioni
  > 20 input →  10 000 campioni

Distribuzione normale: trasformata Box-Muller
  z0, z1 ~ N(0,1) da u1, u2 ~ U(0,1)
  z0 = √(−2·ln(u1)) · cos(2π·u2)
  z1 = √(−2·ln(u1)) · sin(2π·u2)

Output: output[i] = formula(campione_i)
output.sort()
min = percentile(output, tail)       # tail = (100−confidence)/2
max = percentile(output, 100−tail)   # default: p2.5, p97.5

  • Percentuale: ±N% o +A% / −B% (asimmetrica)
  • Tag metodo: WC | RSS | MC
  • Fonte: declared (input) | propagated (formula)

Distribuzione in input

Ogni input può avere una distribuzione associata. Quando la propagazione usa un metodo deterministico (WC/RSS), il grafico mostra la PDF teorica dell'input. Con Monte Carlo viene disegnato l'istogramma della distribuzione finale.

Fallback distribuzione uniforme: Se la distribuzione non è specificata, il motore assume una distribuzione uniforme come default. Questo garantisce che la propagazione sia sempre calcolabile, ma può sovrastimare l'incertezza per input che seguono altre distribuzioni.
