CalcDocs Complete Guide

Formulas YAML + inline calculations for embedded C/C++ projects.

1) YAML Formulas: structure

ADC_MAX:
  type: const
  value: 4095
  unit: count

R_PULLUP:
  type: const
  value: 10
  unit: kohm

NTC_ADC:
  type: expr
  expr: ADC_MAX * NTC_R / (R_PULLUP + NTC_R)
  unit: count
  value: auto

type: const fixed scalar value

type: expr expression with dependencies

type: lookup tabular value from CSV (direct or interpolated)

2) Dependency graph and evaluation order

CalcDocs builds a dependency graph and resolves symbols in topological order.

  • Error if a symbol is undefined.
  • Error if a circular dependency is found.
  • Warning if a formula depends on ambiguous C/C++ conditional symbols.

3) Units, compatibility and dimensional checks

Compatible units can be mixed freely when dimensions match:

PRESSURE_OK:
  type: expr
  expr: 100 bar + 10 kPa
  unit: atm    # valid conversion chain bar -> Pa <- kPa -> atm

Automatic conversion is applied only when strictly necessary: additions/subtractions between compatible units and final conversion between scaled variants of the same unit (for example, V -> mV). Pure multiplication/division across different systems (for example, bar -> atm) keeps the native resulting unit.

Incompatible sums/subtractions are rejected:

BAD_SUM:
  type: expr
  expr: 10 ohm + 3 V   # error

When unit: is omitted, hover shows the dimension-derived resulting unit where possible.

4) CSV / lookup formulas

NTC_R:
  type: lookup
  table: ntc_10k_table.csv
  row: 25
  lookupColumn: temp_c
  valueColumn: resistance_ohm
  interpolation: linear
  unit: ohm

Equivalent expression form:

expr: csv("ntc_10k_table.csv", 25, "temp_c", "resistance_ohm", "linear")

5) Explain mode in hover

Hover includes:

  • computed value and final output unit
  • expanded formula
  • resolved dependencies (with value + unit when available)
  • step-by-step explain chain
= 4095 * 10 / (10000 + 10)
= 4095 * 10 / 10010
= 4.0909090909

6) Inline calculations (C/C++ comments)

// @vin = 12V
// @r = 4.7kOhm
// = @vin / @r -> mA

// = 25% * 200W -> W
// = 100 bar + 10 kPa -> atm
// = 3 ft + 12 in -> m

Inline supports compatible SI/imperial conversions and dimensional diagnostics.

7) Inline ignore directives

// = A + B #calcdocs-ignore-warning
// = BAD_EXPR #calcdocs-ignore-error
// calcdocs-ignore-line; = anything

Available: calcdocs-ignore, calcdocs-ignore-error, calcdocs-ignore-warning, calcdocs-ignore-info, calcdocs-ignore-line.

8) C/C++ unit extraction

#define R_PULLUP 10000 // ohm
const float VREF = 3.3; // V

CalcDocs imports detected units and enriches YAML symbols. Missing YAML entries are suggested in diagnostics.

9) Hover coexistence with other extensions

CalcDocs hover is now constrained to known CalcDocs symbols/macros and inline contexts, to reduce collisions with IntelliSense and third-party providers.

If no CalcDocs-relevant context exists, CalcDocs returns no hover and leaves the UI to other extensions.

10) Recommended workflow

  1. Define constants in C/C++ with inline unit comments.
  2. Create formulas in YAML with type, expr, and optional unit.
  3. Use hover/diagnostics to fix undefined symbols, cycles, and unit mismatches.
  4. Use inline comments for quick one-off checks in source files.
